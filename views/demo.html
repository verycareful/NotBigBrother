Blind Signature Flow

Walk through every step of the NotBigBrother protocol in your browser. All cryptographic operations happen locally — the server only sees a blinded hash, never your token.

1
Fetch Server Public Key
·
Retrieve the issuer's RSA public key components N and E. Anyone can fetch this — it's intentionally public, used to verify tokens offline. 
Waiting…
↓
2
Generate Token Payload
·
Create the token locally in your browser. A unique random nonce ensures every token is distinct. This payload never leaves your device in plaintext. 
Waiting…
↓
3
Blind the Token
·
Multiply the token's SHA-256 hash by a random blinding factor r using the server's public key: blinded = H(token) · rE mod N. The server will sign this — not the original token. 
Waiting…
↓ (only this blinded value is sent to the server)
4
Request Blind Signature
·
Send the blinded value to the issuer. The server runs a mock identity check (always passes in this demo), then signs the blinded hash with its private key. It has no idea what it's signing — only that you passed verification. 
Waiting…
↓ (server's blind signature returned)
5
Unblind the Signature
·
Remove the blinding factor locally: unblinded = blindSig · r-1 mod N. The result is a valid RSA signature on your original token — with zero linkage to this signing session. 
Waiting…
↓
6
Verify Locally
·
Check the signature offline using only the public key: unblindedE mod N = H(token). No server call. No account. This is exactly what any website does when you present your credential. 
Waiting…
↓
7
Save Credential
·
Download your credential as nbb-credential.nbb — a compact, opaque token file. Format: nbb1.<base64url>. Store it like a password. Present it to any site that accepts NotBigBrother tokens; verification is offline, no server needed. 
Waiting…
This is a mock identity check. In production, step 4 requires real age verification (government ID, bank check, etc.) before the server signs anything. The cryptographic flow is identical — only the gating condition changes. The server still learns nothing about where you'll use your token.
